Lab Environmental Monitoring System

Apr 2025 – May 2025
Academic Project (MUET)
Arduino Uno C/C++ I2C Protocol Sensors

Project Overview

In environments sensitive to environmental changes (such as chemical labs, clinical cleanrooms, or high-precision calibration labs), maintaining constant ambient light intensity, temperature, and relative humidity is critical. Even slight deviations can trigger unwanted chemical reactions or corrupt test calibrations.

The Lab Environmental Monitoring System was designed and engineered as a continuous, low-latency monitoring solution. Interfacing high-precision digital sensors with an AVR-based Arduino Uno microcontroller, the system monitors ambient light intensity (lux) and climate parameters, providing real-time local output alerts and preparing data structures for future IoT automation.

Technical Implementation & Architecture

The core architectural flow involves high-frequency sensor reading loops, noise filtering, threshold testing, and real-time visualization:

  • High-Precision Light Sensing: Integrated the BH1750 digital ambient light sensor. Unlike analog photoresistors, the BH1750 directly outputs digital values in Lux over the I2C bus (standard address 0x23), avoiding analog voltage drop errors and noise over longer lines.
  • Microclimate Sensing: Deployed a DHT11 sensor using a custom library protocol to sample relative humidity and temperature periodically, ensuring consistent readings without blocking microcontroller execution.
  • Non-Blocking I2C Bus Management: Configured direct register polling on the Atmega328p hardware I2C peripheral, optimizing the clock speed to 400kHz (Fast Mode) to minimize sensor polling latency.
  • Local Indicator Logic: Set up visual LED alarms that trigger instantly when light levels cross calibrated upper/lower thresholds, ensuring rapid alerts for lab technicians.

Firmware Walkthrough

The firmware is written in C++ for the Arduino platform, prioritizing modular structure and low execution overhead. Below is the simplified core loop handling the I2C bus interface and the logic for the sensors:

firmware.ino C++ (Arduino) 
#include <Wire.h>
#include <DHT.h>

#define DHTPIN 2
#define DHTTYPE DHT11
#define LED_ALERT_PIN 13

// Calibration and Threshold limits
const float LUX_LOW_LIMIT = 100.0; 
const float LUX_HIGH_LIMIT = 1000.0;

DHT dht(DHTPIN, DHTTYPE);

void setup() {
  Serial.begin(9600);
  Wire.begin();
  dht.begin();
  
  pinMode(LED_ALERT_PIN, OUTPUT);
  
  // Configure BH1750 for High Resolution Mode (1lx resolution)
  Wire.beginTransmission(0x23);
  Wire.write(0x10); // Continuously H-Resolution Mode
  Wire.endTransmission();
  
  Serial.println("System Initialized. Sampling started...");
}

void loop() {
  // Read Temperature and Humidity from DHT11
  float temp = dht.readTemperature();
  float hum = dht.readHumidity();

  // Read Lux value from BH1750 over I2C
  int rawLux = 0;
  Wire.beginTransmission(0x23);
  Wire.requestFrom(0x23, 2);
  if (Wire.available() == 2) {
    rawLux = Wire.read();
    rawLux <<= 8;
    rawLux |= Wire.read();
  }
  Wire.endTransmission();
  
  float lux = rawLux / 1.2; // Convert to Lux per datasheet
  
  // Threshold assessment
  if (lux < LUX_LOW_LIMIT || lux > LUX_HIGH_LIMIT) {
    digitalWrite(LED_ALERT_PIN, HIGH); // Alarm ON
  } else {
    digitalWrite(LED_ALERT_PIN, LOW);  // Alarm OFF
  }

  // Log values
  Serial.print("Temp: "); Serial.print(temp); Serial.print(" C | ");
  Serial.print("Hum: "); Serial.print(hum); Serial.print(" % | ");
  Serial.print("Light: "); Serial.print(lux); Serial.println(" lx");

  delay(1000); // Sample rate limit
}
